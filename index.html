获取Let's Encrypt免费SSL证书

发表于 2018-08-24 | 分类于 运维

1.SSL证书

1.1 获取证书

获取Let’s Encrypt免费SSL证书，首先获取脚本:

git clone https://github.com/letsencrypt/letsencrypt
cd letsencrypt
chmod a+x certbot-auto

使用Certbot客户端申请证书(注意，有可能ngixn启动导致80端口被占用无法获取，建议先停止nginx):

./certbot-auto certonly  -d "*.shandudata.com" --manual --preferred-challenges dns-01  --server https://acme-v02.api.letsencrypt.org/directory

添加完DNS解析后回车继续执行，就能在/etc/letsencrypt/live/shdandudata.com中看到生成的证书和密钥。

3.2 nginx配置证书

在nginx配置文件中添加对上面生成的文件的引用就能使用ssl证书，如下是例子:

server {
    listen 443 ssl;
    server_name jumpserver.shandudata.com;
    ssl on;
    ssl_certificate /etc/letsencrypt/live/shdandudata.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/shdandudata.com/privkey.pem;
    ssl_trusted_certificate  /etc/letsencrypt/live/shdandudata.com/chain.pem;
    ssl_session_timeout 5m;
    ssl_protocols TLSv1 TLSv1.1 TLSv1.2;
    ssl_ciphers AESGCM:ALL:!DH:!EXPORT:!RC4:+HIGH:!MEDIUM:!LOW:!aNULL:!eNULL;
    ssl_prefer_server_ciphers   on;

    ……
}

3.3 证书续期

Let’s encrypt 的免费证书默认有效期为 90 天，可以添加crontab自动获取最新证书:

0 */12 * * * /home/deploy/github/letsencrypt/certbot-auto renew --renew-hook "/etc/init.d/nginx reload"

Jumpserver安装与坑

发表于 2018-08-24 | 分类于 运维

1 Jumpserver安装

安装过程见jumpserver官网文档

jumpserver官方安装文档

个人虚拟环境配置采用virtualenvwrapper + pyenv

2.Jumpserver踩过的坑

  1. Web页面显示没有正确渲染

    通过Nginx代理jumpserver，查看Nginx后台日志报错如下:

    2018/07/30 09:35:47 [error] 54615#54615: *4 open() "/home/root123/jumpserver/dat
a/static/css/plugins/toastr/toastr.min.css" failed (13: Permission denied), clie
nt: 192.168.0.101, server: _, request: "GET /static/css/plugins/toastr/toastr.mi
n.css HTTP/1.1", host: "192.168.0.100", referrer: "http://192.168.0.100/"

    原因为nginx默认配置中采用的是www-data用户启动，会造成读取jumpserver静态文件时没有权限，页面无法加载静态文件而报错。修改nginx默认配置文件／etc/nginx/nginx.conf开头使用正确的用户:

    user root;
worker_processes auto;
#pid /run/nginx.pid;

  1. jumpserver测试资产连通性失败

    测试jumpserver资产连通性时失败，页面报错如下:

    fatal: [192.168.0.170]: FAILED! => {"msg": "to use the 'ssh' connection type with passwords, you must install the sshpass program"}

    需要在服务器上安装sshpass:

    apt-get -y install sshpass

  2. jumpserver管理用户

    默认情况下jumpserver推荐管理用户是资产上的root，但为了安全考虑服务器的root用户全部禁用了ssh登陆（有可能被人采用暴力破解掉root密码）。可以新建一个具有sudo权限的用户，在/etc/sudoers中为该用户添加如下:

    xxxxx ALL=(ALL) NOPASSWD: ALL

Ansible安装及部分坑

发表于 2018-08-23 | 分类于 运维

1.Ansible基础知识

1.1 Ansible安装

1.1.1 Mac下安装

在Mac系统中安装Homebrew，之后使用brew安装ansible:

ruby -e "$(curl --insecure -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"
brew update
brew install Ansible

1.1.2 Ubuntu下安装

为Ubuntu获取Ansible的最佳方法是将项目的PPA（个人包存档）添加到系统中:

sudo apt-add-repository ppa:ansible/ansible

接下来刷新系统的包索引并安装ansible:

sudo apt-get update
sudo apt-get install ansible

1.2 Ansible配置

2.Ansible踩过的坑

  1. 目标主机python版本

    在ansible的配置文件hosts中存放的目标主机的ip或域名，ansible连接过去后默认采用的是/usr/bin/python执行命令，可能会出现目标主机未安装python2，只安装了python3导致没有/usr/bin/python,执行ansible命令失败，有两种解决方法：

    A. 在目标主机生成/usr/bin/python软连接，可通过安装python2或者连接python3到/usr/bin/python

    B. 修改hosts文件,为目标主机指定固定的python版本:

    [Client]
192.168.0.170 ansible_python_interpreter=/usr/local/bin/python3
192.168.0.174 ansible_python_interpreter=/usr/local/bin/python3
192.168.0.176 ansible_python_interpreter=/usr/local/bin/python3
192.168.0.172 ansible_python_interpreter=/usr/local/bin/python3
192.168.0.178 ansible_python_interpreter=/usr/local/bin/python3

  2. 批量分发密钥

    批量分发密钥脚本内容如下:

    push_key.yml
---
- hosts: all
  remote_user: "{{ user }}"

  tasks:
    - name: ensure path exist
      file: path={{ ansible_env.HOME }}/.ssh state=directory owner={{ user }} group={{ user }} mode=0700

    - name: push ssh-key
      copy: src=authorized_keys dest={{ ansible_env.HOME }}/.ssh/authorized_keys owner={{ user }} group={{ user }} mode=0644

    调用方式：ansible-playbook push_key.yml -e “user=bigdata” –ask-pass
    注意这里参数–ask-pas(简写-k)s为用户对应的密码，不是sudo密码，sudo密码参数是–ask-become-pass(或者-K)

  3. source

    ansible-playbook脚本中shell和command模块可以执行shell命令，但source命令会执行失败，报没有该命令，目前还未找到解决方法。

  4. pip问题

    这个不算ansible的坑，只是在执行ansible批量安装时发现的。
    ubuntu安装pip，网上一堆教程说采用apt-get install python-pip，但是这种方法安装的pip版本是8.1.1，用这个pip安装其他包有可能会报版本过低，一般会采用pip install –upgrade pip更新pip，然后pip就有可能报一些奇奇怪怪的错误::

    Traceback (most recent call last):
    File "/usr/local/bin/pip", line 7, in <module>
    from pip._internal import main
ImportError: No module named _internal

    建议不要采用上述方法安装pip，如果已经使用了上面的方法，请先卸载pip::

    apt-get remove python-pip

    正确安装pip的方法如下::

    wget https://bootstrap.pypa.io/get-pip.py
python get-pip.py
pip install --upgrade pip

gitlab安装与邮箱配置

发表于 2018-08-23 | 分类于 运维

1.docker

由于一步步安装gitlab比较麻烦，这里推荐通过docker部署gitlab

1.1 开始安装

由于apt官方库里的docker版本可能比较旧，所以先卸载可能存在的旧版本:

sudo apt-get remove docker docker-engine docker-ce docker.io

更新apt包索引:

sudo apt-get update

安装以下包以使apt可以通过HTTPS使用存储库（repository）:

sudo apt-get install -y apt-transport-https ca-certificates curl software-properties-common

添加Docker官方的GPG密钥:

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -

使用下面的命令来设置stable存储库:

sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"

再更新一下apt包索引:

sudo apt-get update

安装最新版本的Docker CE:

sudo apt-get install -y docker-ce

1.2 验证docker

查看docker服务是否启动:

systemctl status docker

若未启动，则启动docker服务:

sudo systemctl start docker

添加开机自启动:

sudo systemctl enable docker

2.gitlab

2.1 安装

安装命令:

sudo docker run --detach \
    --hostname 45.40.248.73 \
    --publish 10443:443 --publish 10080:80 --publish 10022:22 \
    --name gitlab \
    --restart always \
    --volume /home/deploy/docker/gitlab/config:/etc/gitlab \
    --volume /home/deploy/docker/gitlab/log:/var/log/gitlab \
    --volume /data/gitlab/data:/var/opt/gitlab \
    gitlab/gitlab-ce:latest

参数说明:

  • –detach: 设置容器后台运行
  • –hostname: 设置容器的 hostname
  • –publish: 端口转发规则（10080: Http 访问端口，10443: Https 访问端口，10022: Docker 容器中 ssh 访问端口）
  • –name: 容器名称
  • –restart always：每次启动容器就重启GitLab
  • –volume: 共享目录挂载，即 docker 容器内外数据共享（/home/deploy/docker/gitlab/config: GitLab 的配置文件，/home/deploy/docker/gitlab/log: GitLab 的 log，/data/gitlab/data: 应用程序数据）

2.2 邮箱配置

修改gitlab配置不建议直接修改/home/deploy/docker/gitlab/config/gitlab.rb。建议进入docker容器中修改配置:

sudo docker exec -it gitlab bash
vim /etc/gitlab/gitlab.rb

修改的邮箱配置如下:

gitlab_rails['smtp_enable'] = true
gitlab_rails['smtp_address'] = "smtp.exmail.qq.com"
gitlab_rails['smtp_port'] = 465
gitlab_rails['smtp_user_name'] = "XXXXX@shandudata.com"
gitlab_rails['smtp_password'] = "XXXXX"
gitlab_rails['smtp_domain'] = "shandudata.com"
gitlab_rails['smtp_authentication'] = "login"
gitlab_rails['smtp_enable_starttls_auto'] = true
gitlab_rails['smtp_tls'] = true

gitlab_rails['gitlab_email_from'] = 'XXXXX@shandudata.com'

修改完后需要重新加载配置:

gitlab-ctl reconfigure
gitlab-ctl restart

测试邮箱是否设置成功:

gitlab-rails console
ActionMailer::Base.delivery_method
ActionMailer::Base.smtp_settings
Notify.test_email('xxx@shandudata', 'Message Subject', 'Message Body').deliver_now
